Project Overview

Use SQL, object-oriented programming (OOP), APIs, and NoSQL databases to analyse the data. This includes performing exploratory data analysis (EDA), retrieving weather data for games played in 2011 via an API, and finally exporting the enriched dataset into a NoSQL database.

I calculated the number of goals scored, as well as the wins, draws, and losses for each team during the 2011 season. The analysis showed that most teams in the dataset were from the Bundesliga. When enriching the dataset with weather data via an API, we observed a high number of null values and very few rainy matchdays. As a result, calculating the percentage of wins in rainy conditions relative to total wins would have produced negligible and misleading results. Instead, we chose to present the number of wins, losses, and total games played in rainy conditions for each team.

Key Features

Demonstrated analytical judgement by adapting metrics based on data quality

Identified high null values in weather data and adjusted analysis approach to present wins/losses in rainy conditions instead of calculating misleading percentages.

Applied a hybrid SQL–Python workflow to handle complex data structures

Due to the quality of the data I Correctly determined that percentage-based win metrics would be misleading. Instead, you reported absolute counts of wins, losses, and total rainy matches, demonstrating statistical reasoning, data quality assessment, and responsible interpretation.

Built and optimised an API data-enrichment pipeline under real-world constraints

I successfully enriched the dataset with external weather data using the DarkSky API, while working within a strict request-limit (1,000 calls/day). By caching results to CSV, you avoided redundant requests and ensured reproducibility and efficiency

Designed an end-to-end data pipeline using both relational and NoSQL databases

Completed the workflow by exporting question-specific datasets and visual outputs into MongoDB using PyMongo, showcasing the ability to work across relational (SQL) and NoSQL paradigms.
